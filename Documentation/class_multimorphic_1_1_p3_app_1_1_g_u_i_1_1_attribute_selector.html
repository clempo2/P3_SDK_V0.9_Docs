◆ OnReceiptOfDataFromMode()

override void Multimorphic.P3App.GUI.AttributeSelector.OnReceiptOfDataFromMode ( object dataFromMode)
protectedvirtual

This method is called when the selector receives data that was sent via a mode calling SetSelectorData. A mode may set the data before this selector is enabled, so SelectorManagerMode will hold the data until this selector gets enabled, at which time this method will be called. Override this method to interpret/restructure the data and/or store it locally so that it can inform the PrepareDataObjects method later. The contents of dataFromMode will also be stored in this.dataReceivedFromMode for later reference.

IMPORTANT: When overriding this method, call base.OnReceiptOfDataFromMode at the end of the method, not the beginning.

In most cases, the overridden version of this method will need to call Clear() because the new data replaces the old data. This must happen before calling the base method.

Parameters
dataFromModeThe data that the mode considered relevant to this selector.

Reimplemented from Multimorphic.P3App.GUI.Selector.Selector.

◆ selectorItemPrefab

UnityEngine.Object Multimorphic.P3App.GUI.Selector.Selector.selectorItemPrefab
inherited

Represents how many items from the end of the visible items scrolling should commence. This allows the next invisible items to become visible before they're selectable.
More importantly, margin can be used to create a list that scrolls around a single centrally selectable item, by setting margin to {visibleCount-1) / 2 and ensuring visibleCount is an odd number.

The prefab object which will be instantiated for each selectable item. This prefab must have a SelectorItem component (or descendant).
