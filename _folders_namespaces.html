Folders and Namespaces

The Assets Folder Structure

The Assets folder in the Unity project has been structured to help organize P3 app development, to keep the modes and UI elements separate, and to keep the framework separate from app-specific code. Some of the framework code depends on this structural convention, so it is best to adhere to the existing hierarchy.

Folder Contains
Assets/Scripts/Modes Scripts which implement the Mode layer
Assets/Scripts/Modes/{scene name} Scripts which implement the Mode layer for the given scene
Assets/Scripts/GUI Scripts which implement the GUI layer
Assets/Scripts/GUI/{scene name} Scripts which implement the GUI layer for the given scene
Assets/Resources/Prefabs/Framework Prefabs which are part of the P3 framework
Assets/Plugins DLLs which form the P3 framework

You should never add or modify prefabs in the Assets/Resources/Prefabs/Framework folder.

The Namespace Structure

The Multimorphic namespace has been structured to help organize P3 app development and keep the mode (game logic) and UI code (GUI) separate. Your app will have its own namespace but should follow the general structure of the P3SA.

Namespace Responsibility
Multimorphic.P3 Platform internals
Multimorphic.P3App.Modes Modes that are part of the P3 framework
Multimorphic.P3App.GUI GUI code that is part of the P3 framework
Multimorphic.P3App.Modes.Data Data management modes that are part of the P3 framework
Multimorphic.P3App.Modes.Menu Modes which support menuing as part of the P3 framework
Multimorphic.P3App.GUI.Menu GUI code to support menuing as part of the P3 framework
Multimorphic.P3SA.Modes Modes that implement the application sample, P3SA
Multimorphic.P3SA.GUI GUI code for the application sample, P3SA
YourCompany.YourAppCode.Modes Your app-specific modes
YourCompany.YourAppCode.GUI Your app-specific user interface
