Getting Started

Installation

Follow these steps to install the P3 SDK.

  1. Download and unzip the P3 SDK into a convenient directory.
  2. Download and install Unity 5.6.7, see below.
  3. Complete the installation by following the instructions in the P3 SDK's INSTALL.txt file.

Unity Installation

You will need to install Unity on your development machine (See www.unity.com). Unity 5.6.7 is recommended at the time of this writing.

Unity 5.6.7 is available for Windows. Download from here.

It is important to understand some fundamentals of Unity development before proceeding. The Unity website provides excellent learning material plus there are many independent tutorials to be found on YouTube.

At minimum, you should have an understanding of:

  • GameObjects
  • Components
  • The role of the Awake, Start and Update methods
  • Scenes
  • Writing C# scripts in Monodevelop to be added to GameObjects

If your app will want to have the physical pinball interact with virtual screen elements, it will also be important to understand:

  • Rigidbody and Collider components
  • Collision detection

After installing Unity, do not forget to continue the installation by following the instructions in the INSTALL.txt file.

The P3 Application Sample

The framework comes with a working (albeit simple) sample application. It has an attract mode, a playable game (multi-player), settings management and profile management. You can use it as reference for best practices for P3 app implementation. More importantly, it is used as the base starting point for your own app.

To open the P3SA project, launch Unity and select the Open Project option. When prompted, select the top-level folder of the unzipped application to load the project and all assets. Open one of the scenes by navigating to the Scenes folder in the assets hierarchy. Double click on Bootstrap, Attract or Home to view those scenes in the editor.

  • Bootstrap is the scene which initiates the app.
  • Attract is the scene that appears when a game is not being played.
  • Home is the first scene that the player sees when a game commences.

The Simulator

To start your app in the Editor's Game window, make sure the Bootstrap scene is loaded, then press the play button (â–º).

Note: It is a common error to try to play the app in the Editor with another scene than Bootstrap loaded. Doing this causes strange errors because the app is not properly initialized.

There are some key mappings to simulate the buttons on the cabinet:

Key Simulates
S Start button
L Launch
A Left flipper
; Right flipper
Q Left upper button
Z Left lower button
P Right upper button
/ Right lower button
0 Ball drain

These mappings (and more) are defined in the configuration file P3SampleApp/Configuration/AppConfig.json

Note: Unity listens for the key code, which corresponds to the physical keys on the keyboard, regardless whether the Shift key is held. For example, pressing the A key means typing lowercase a.

If you click and drag the mouse in the Game window, a red trail follows the mouse. This is the in-Unity simulation of the physical ball. It's the ball's avatar in the Unity scene, and it extends as far as the scene camera can see. It has a rigid body and collider, so collisions can be detected. On the P3 machine, the actual ball will have the same effect in the scene, minus the visible red trail.
