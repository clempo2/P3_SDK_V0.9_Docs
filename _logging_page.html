Logging

Creating log messages

The framework includes a thread-safe logger which is a very useful tool for determining what's happening in your app. The output goes to the Unity console and log file. The C# syntax for logging safely in your P3 app is:

Multimorphic.P3App.Logging.Logger.Log("An interesting message");
Multimorphic.P3App.Logging.Logger.LogWarning("An important message");
Multimorphic.P3App.Logging.Logger.LogException(exception);
Multimorphic.P3App.Logging.Logger.LogError("A notification of a problem or unexpected result");

For colored logging in the Unity console, add a category to the logging message from Multimorphic.P3App.Logging.LogCategories.

Multimorphic.P3App.Logging.Logger.Log(LogCategories.Game, "A game-specific logging message");

Viewing log messages

When simulating your app in the Unity Editor, you can view your logging messages in the Console window.

When running your app on the P3, you can retrieve the latest 10 logs from the machine in System Manager -> Manage Software -> Manage Apps -> <your app> -> Copy logs to USB

Note - The P3 framework provides a GameAttribute (operator setting) to set the logging level (Settings -> Info -> Logging Level). This guides which messages are actually written to the logs. When simulating in the Unity Editor, this GameAttribute defaults to "All". On the P3, in order to not have a long-running machine consume the SSD with verbose logging, it defaults to "Exception". When debugging apps running on the P3, be sure to change the operator setting to "All" before activating whatever features you're trying to debug.

Note - In the Unity Console, there are buttons for hiding each of Log/Warning, Exception and Error log entries. So if you're wondering why certain messages aren't appearing, make sure that the console window isn't set to hide them.
